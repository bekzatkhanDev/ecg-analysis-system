# ECG Analysis Frontend

## Обзор

Frontend часть системы анализа ЭКГ, построенная на React с TypeScript, Tailwind CSS и современными подходами к разработке.

## Особенности

- 🎨 Современный интерфейс с Tailwind CSS
- 🔐 JWT аутентификация и управление сессиями
- 📊 Интерактивная визуализация результатов анализа ЭКГ
- 📱 Адаптивный дизайн для всех устройств
- 🔄 Реактивное управление состоянием с Zustand
- 📈 Прогресс-бары загрузки и анализа
- 📁 Поддержка多种 форматов файлов (JSON, CSV, TXT)

## Требования

- Node.js 18+
- npm или yarn
- Modern browser с поддержкой ES2020+

## Установка

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd ecg-analysis-system/frontend
```

2. Установите зависимости:
```bash
npm install
# или
yarn install
```

3. Настройте окружение:
```bash
cp .env.example .env
```

## Настройка

### Переменные окружения

Создайте файл `.env` с следующими переменными:

## Использование

### Аутентификация

1. **Регистрация**: Перейдите на `/login` и выберите "Register"
2. **Вход**: Введите email и пароль для входа в систему
3. **Сессия**: Токен автоматически сохраняется и используется для всех запросов

### Загрузка ЭКГ

1. **Выбор файла**: Нажмите "Choose File" и выберите файл ЭКГ
2. **Поддерживаемые форматы**:
   - JSON файлы с данными ЭКГ
   - CSV файлы с разделителями (запятая, точка с запятой, пробел)
   - TXT файлы с числовыми данными

3. **Требования к данным**:
   - 12 отведений ЭКГ
   - 5000 образцов на каждое отведение
   - Общее количество: 60,000 значений

### Анализ ЭКГ

1. **Загрузка**: После выбора файла нажмите "Start Analysis"
2. **Прогресс**: Отслеживайте прогресс загрузки и анализа
3. **Результаты**: Просмотрите вероятности для каждого класса:
   - NORM: Нормальный ритм
   - MI: Инфаркт миокарда
   - STTC: Ишемические изменения
   - CD: Нарушение проводимости
   - HYP: Гипертрофия миокарда

## API Интеграция

### Аутентификация
- JWT токены автоматически добавляются ко всем запросам
- Хранение токена в localStorage
- Автоматическая проверка сессии

### Запросы к API
- **POST /api/auth/login** - Вход пользователя
- **POST /api/auth/register** - Регистрация пользователя
- **GET /api/users/me** - Получение данных пользователя
- **POST /api/ecg/analyze** - Анализ ЭКГ сигнала

### Обработка ошибок
- Централизованная обработка ошибок API
- Пользовательские сообщения об ошибках
- Автоматическое восстановление после ошибок
